## 2026-07-21 - [Accessible Client-Side Copy Interactions and State Caching]
**Learning:** For client-side copy interactions (such as the ClipboardCopy hook), screen readers must be informed of status changes politely using a visually hidden `aria-live="polite"` element and dynamic `aria-label` updates. Caching the original `aria-label` and label texts on mount is critical to prevent state corruption from rapid successive clicks, ensuring the interface gracefully restores to its original state.
**Action:** Always capture and store original interactive labels (e.g., `aria-label` and inner text labels) upon component mount, set up a persistent or dynamically managed `aria-live` element, and design the timeout restoration logic to handle concurrent clicks without losing the original state values.
