## 2026-03-06 - [Accessible Clipboard Copying Feedback Pattern]
**Learning:** Client-side copying is typically an invisible interaction to screen reader users because changes to button labels ("Copy" to "Copied") do not automatically trigger announcements. By appending a visually hidden (`sr-only`) element with `aria-live="polite"` inside the interactive element, we can politely announce the state update. Additionally, caching the original `aria-label` on mount prevents corruption of the focus/interaction labels during rapid, successive copy clicks.
**Action:** Always implement a dedicated visually hidden screen-reader announcement element and cache original hover/announcement attributes on mount when building highly interactive client-side components like copy buttons or toggles.
